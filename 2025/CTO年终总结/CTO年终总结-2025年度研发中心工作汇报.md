## 一、年度核心成果

### 1.1 Drive产品战略突破

**Drive团队从零组建，半年交付3个大型企业客户**

| 客户 | 交付内容 | 亮点 |
|------|----------|------|
| 广电 | 生产环境部署、所有者功能、权限回收、去品牌化 | 数据迁移从6-8小时缩短至30分钟 |
| 好未来 | 1.6版本升级、密级标签系统、JSSDK定制 | 全流程密级管控 |
| OPPO | CO1.5版本交付、群组功能、折叠屏适配、暗黑主题 | 双端完整产品 |

**Drive的战略意义：**
- Drive替代原有复杂全站架构，大幅降低部署和运维成本
- 利润可观，商业化可大范围推广
- 我们的愿景：成为**"文档界的GitLab"**，让大部分企业都能用上私有化的石墨文档

### 1.2 私有化交付成果

- 完成**17个大版本升级**：滴滴、TCL、OPPO、跨越速运、新华三、喜马拉雅、格力、盛天网络、招商金科、小红书、广东电信研究院、微众银行、中广核、云鲸智能、长亮科技、中船704等
- 新增**13个正式交付**：中电信数智、武汉胜意、天翼云盘、福昕网络、百度文心一言、卡斯柯等
- **15个POC项目**：北京钢研、岚图汽车、海信集团等

### 1.3 内部体验机制建立（Dogfooding）

**这是2025年最重要的流程变革之一。**

**之前的问题：**
- 我们没有真正自己体验私有化的全站和SDK产品
- 客户提出的SDK API和体验问题，没有得到足够重视

**改变措施：**
- 引入Drive团队，通过Drive产品使用SDK
- 构建shimo.net网站，让内部团队真正使用Drive和SDK产品
- 通过私有化升级方式处理Drive，把私有化问题暴露到内部使用环境

**关键发现：**
- Drive使用SDK后，验证了客户反馈的问题都是真实的
- 例如：协作头像显示等体验问题
- 站在客户角度才能真正理解痛点

**效果：**
- 更早发现问题，更容易解决客户问题
- 减少客户侧返工

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|

### 2.3 极速SDK与自动化部署

**极速SDK：**
- 单二进制部署，无MySQL/Minio依赖
- 完成讯飞交付，支持docker部署
- 可公开下载，License控制水印

**自动化部署（mdp-installer）：**
- POC部署无需立项，非SRE人员可部署
- 大幅降低部署、升级、运维成本
- 为商业化大范围推广奠定基础

### 2.4 轻文档导出优化

| 指标 | 优化前 | 优化后 | 节省 |
|------|--------|--------|------|
| 内存占用 | 44GB | 13GB | **31GB** |

- 替换officeio，稳定性大幅提升

## 八、问题与挑战

### 8.1 需要反思的问题

| 问题 | 说明 |
|------|------|
| 客户反馈重视不够 | 之前对客户提出的SDK体验问题没有足够重视，Dogfooding后验证客户是对的 |
| 代码问题占比高 | Bug的69.3%为代码问题，需加强Code Review和单元测试 |
| 人员单点风险 | 每个研发仅负责一个套件，请假/离职导致项目停滞 |

### 8.2 持续面临的挑战

| 挑战 | 应对 |
|------|------|
| 专业文档功能不齐 | 功能覆盖不如金山，影响商业竞争力 |
| 幻灯片H5编辑缺失 | 移动端不支持编辑，客户反馈强烈 |
| 交付持续消耗 | 占总工时20%+，需标准化流程，建立自助能力 |

## 九、2026规划展望

### 9.1 战略背景变化

| 驱动力 | 核心变化 | 战略影响 |
|--------|----------|----------|
| 资金缓解 | Z项目资金到位 | 从"求生存"转向"求发展" |
| 产品架构 | Drive替换复杂全站 | 商业化可大范围推广 |
| 国际化机遇 | 俄罗斯UCCP项目 | 类飞书产品、应用市场、海外私有化 |
| 运维能力 | CO1.5自动化工具 | 非SRE可部署，POC无需立项 |

### 9.2 核心战略目标

1. **商业化**：成为"文档界的GitLab"，Drive大范围推广
2. **产品**：补齐套件功能短板，SDK/JS API全面增强
3. **技术**：协同编辑Go化、轻文档重构、产品国际化

### 9.3 组织架构调整

采用"Drive + SDK + SDK Core"三层架构：

| 层级 | 职责 | 优势 |
|------|------|------|
| **Drive** | 最外层，负责产品体验和商业化 | 聚焦核心场景和卖点 |
| **SDK** | 胶水层，统一前后端逻辑 | 统一私有化对接团队 |
| **SDK Core** | 纯套件，专注UI库和JS API | 打造高可用JS SDK API |

## 十、结语

2025年，研发中心在客户交付、技术突破、流程优化方面取得了实质性进展。

**最重要的三个变化：**

1. **Drive产品战略落地**：从零组建团队，半年交付3个大客户，证明Drive替代全站架构的可行性
2. **内部体验机制建立**：通过Dogfooding验证客户痛点，改变了"不重视客户反馈"的问题
3. **自动化部署能力**：POC无需立项，为商业化大范围推广奠定基础

2026年，我们将继续以客户为中心，落地"Drive + SDK + SDK Core"三层架构，向"文档界的GitLab"目标迈进。
